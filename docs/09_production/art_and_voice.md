# 立ち絵と口調

## 画風の確定（2026-06-19）

ChatGPT Images2 のデフォルトで出した10分割キャラシートは没。理由＝テンプレ過ぎて個性なし／輪郭・目がかぶる／kako-jun っぽくない。これを次の方針で作り直す。

### 軸：媚びの度合い（三極のうち「中間」を採る）

- 左＝kako-jun 寄り（媚び最小／個性で人を選ぶ）／中＝作家性はあるが媚びない／右＝今風・媚び（水星の魔女的／堀を自分で埋める）。
- **採用：個性は「表情」に、間口は「今風・繊細塗り」に分業する中間。** 「こびない」は本来*塗り*でなく**表情と仕草**の話（カメラ目線で微笑まない・没頭）なので、塗りが今風でも堀は減らない。むしろ塗りで間口を下げるのはシリーズの旗「わかりにくいものをわかりやすく」に沿う。
- ガードレール：**表情を、きらめき・カメラ目線の微笑みまで今風に振らない**。そこまで媚びると水星の魔女に溶けて堀が消える。

### 病名は「平均化（mode collapse）」

つまずいた3つは同じ病気：①ChatGPT 10分割1枚 → 顔が平均化＝目がかぶる ②全キャラを集めた統合 LoRA → いまいち（混ぜると真ん中に落ちる）③10人がイケメン男ばかり → 一つの男前に収束。**処方＝混ぜない＋わざとバラついた素体を与える。**

### 表情の素体：ToHeart 女キャラ8人の「目の形パレット」

8人にツリ目・垂れ目…が**均等に散っている**のを「目の形のパレット」として借用。女性キャラの目の作りを*青年*に移すと**中性的かつ個性的**になり、男前一辺倒（BL/ホスト文法）から自然に外れる。

- 割り当て：**住人8 ↔ ToHeart 8人を 1:1**。せおはやみ（最もプレーンな目）とヴィンチア（司会）はこの対応の外＝別枠。
- **個別 LoRA（8体）＞ 統合 LoRA。** 実測根拠＝「マルチ単体 LoRA は再現度が高い／全部集めた統合はいまいち」（Rog 既存）。統合に戻すのは平均化病の再発。

### IP 衛生

LoRA は**「元キャラだとバレる手前」で止め、原案の中性青年に溶かす**。目の形を“借りる”のはセーフ、キャラを再現すると Leaf/AQUAPLUS の版権を踏む。人格化名で estate を回避したのと同じ勘で、ToHeart の影響も原案に溶かす。

### 制作パイプライン（2段）

- **Stage1（gpt-image-2 / studio-yokonami）**：今風・繊細塗りの中性青年ベースを生成。**版権名（マルチ/ToHeart）は出さない**（拒否・平均顔回避。マルチっぽさは LoRA の仕事）。
- **Stage2（SD ＋ 個別 LoRA / Rog・studio-yokonami）**：ベースに LoRA を**控えめに**乗せ、目・表情の個性を足す。
- **唯一の未知**＝Stage1 の塗りが Stage2 で死なずに残るか。**1キャラ目の実験で潰す**。崩れるなら退路（全 SD ／別手）。
- **サインオフはライブ blink / A-B**（静止画 IoU での「同じ」判定は不可。session594 の教訓）。

### 第1実験と決定ゲート

- **主役せおはやみから。** 既存の**マルチ LoRA で「マルチっぽい中性顔」**に寄せる（柔らかく素直＝聞き役の主役に本採用でもよい配役）。新規学習ゼロで最大の未知を潰せる。
- **決定ゲート**：手応えが出れば「他7体の ToHeart LoRA を作る価値あり」と判断し住人へ展開。出なければ退路へ。
- 進捗・試行錯誤は **studio-yokonami の Issue** に記録する。
- **LoRA 合成前の前提**：せおはやみと同程度のキャラクターシートを全キャラに先に作る。全身・表情・三面図・衣装小物・問いの核を揃えてから LoRA を当てる。仕様は `../07_visual/reference/CHARACTER_SHEET_SPEC.md`。

### キャラクターシート ＝ 原稿スキルの参照正本

各キャラの**立ち絵を作るときに、口調・口癖も同時に確定**する（下記「口調」参照）。**絵＋口調＋口癖＋能力名を束ねた「キャラクターシート」が、将来の原稿生成スキルの参照元（正本）**になる。osaka-kenpo がデータを固定してスキルを決定論にしているのと同じ＝シートを先に固めれば原稿スキルがブレない。順番はシート → スキル。

## 立ち絵

- **顔の特徴を残したまま全員同い年の青年。** 老賢人の蝋人形でなく「同じゼミの、やたら優秀で生意気なクラスメイト」。衣装は時代を匂わせる程度（フルコスプレ・厳粛な肖像画はしない）。
- **"こびない格好良さ"＝誘惑でなく没頭。** カメラ目線で微笑まない、議論に没頭してこちらを見ていない、撮られたと気づかない自然さ。✗きらめき/薔薇/乙女ゲー光彩/はだけ/思わせぶり/ペアの匂わせ/BL・ホスト文法。○地に足のついた配色、理想化しすぎない体型、考え込む・反論する表情。
- **これが BSD（文豪ストレイドッグス）／偉人擬人化に追えない一線。** 媚びる前提のジャンルは「媚びない自然さ」を構造的に追えない＝堀。客層も腐女子ニッチでなく「媚びられるのに飽きた層」へ。**商品はキャラでなく中身、重心は脚本に置く。**
- **せおはやみ**は中性・無色・控えめ＋盲目（天才陣を食わない読者の目）。

## 非イベント絵・サイト/動画用イラスト

キャラ別の高品質イラストはイベントCGではない。実際に起きる場面の再現でなくてよい。キャラの思想、魅力、表情、思考状態を見せる promotional / symbolic illustration として作る。

- 住人は食べなくても存在できる思念体に近いが、悟り切ってはいない。せおに対しては先輩だが、彼ら自身も悩み、研究し、読み直している。
- 各キャラ専用の個室はない。図書館のさまざまな場所で、読書、思索、歩行、議論、沈黙をしている絵にする。背景は「そのキャラの部屋」ではなく「その瞬間の思考状態を映す図書館の場所」。
- 喜怒哀楽、ポーズ、微細演技（涙がつーと伝う、泣くのをこらえる、静かな怒り、答える前に視線を外す等）を含める。
- サイトのカード/ヒーロー/OGP、記事画像、動画生成 reference に使う。話者ターン用の立ち絵・ポーズ差分とは別ラインで管理する。

## 本編イベント絵/物証アップの差し込みメモ（2026-07-16）

生成絵の管理は GitHub Issue #124 に集約する。Issue 数は節約し、1枚ごとの個別 Issue は切らない。プロンプト、参照画像、採用ファイル、再生成理由は #124 の本文表とコメントで追う。

第一幕の追加絵は採用済み:

- `act1-01`: `assets/images/story/act1/star-sea-entrance.webp`
- `act1-03`: `assets/images/story/act1/eleven-chairs-nameplate.webp`
- `act1-04`: `assets/images/story/act1/return-slot-untitled-book.webp`

ただし本文への差し込みは、name-name 側のイベント絵/スチル仕様が固まってから行う。関連は name-name #351。イベント絵は「テキストより背面、背景・立ち絵より前面に出る、画面ぴったり画像レイヤー」として扱う方針。

差し込み時の背面処理:

- `背面=hide`: 本物のイベント絵。背景・立ち絵を消し、イベント絵終了後に必要なら再フェードインさせる。既定はこちら。
- `背面=keep`: 物証アップや一時ズーム。背景・立ち絵は裏で維持し、前面画像だけを出して消す。

Theo の使い分けは、第二幕以降のイベント絵は原則 `背面=hide`、第一幕の物証アップは `背面=keep` 候補。ただし初期実装が `背面=hide` だけなら、第一幕の物証アップ差し込みは後回しにしてよい。本文より先に、イベント絵の状態管理・save/load・seek・deep-link・video export の復元が破綻しないことを優先する。

### 叡智タグ（生成・資産管理用）

`叡智` は、サイト文言では「wisdom / philosophical depth」として使える一方、生成AI文脈では露出高め・紳士向け表現の隠語としても機能する。二重意味を意図的に使うが、公開コピーと生成タグを混線させない。

- 生成・資産管理では `叡智` を、シャツ前開け、ゆるい襟元、鎖骨・胸元の見える構図などの高露出バリアントを指す内部キーワードとして使ってよい。
- 目標は上品なファンサービス／キャラ魅力であり、露骨なポルノ表現ではない。
- 全キャラに機械的に適用しない。neutral / serious / emotional / symbolic / promotional と並ぶ、1つのバリアント系列として扱う。
- せおは中性・読者の分身であり、幼く見える／弱く見える方向の高露出は避ける。叡智バリアントは成人的に見える住人を中心に検討する。
- 公開サイトの本文で `叡智` を使う場合は、基本は「哲学的な叡智」と読ませる。ダブルミーニングを狙う場合だけ意図して使う。

## 口調（セリフの温度）

**口調の基準：「完成した故人」でなく「今も現役で、考えるのを楽しんでいる生きた頭」。**

- ✗「〜であるぞ」式の古文翻訳調 ✗ 教科書朗読の棒読み ✗ 達観した老人の説教
- ○ 切れ味＋軽口。基準温度は「論破か。いいぞ、嫌いじゃない」級。
- 各人を相対的にズラす：カンティア=生真面目で理屈っぽい／ヒュー=皮肉屋で砕け／デカリス=何でも疑う神経質な切れ者／マキヤ=醒めた実務家／オウ=行動を迫る／ヘグル=壮大に巻き込む／スピノ=感情を冷静に解剖／アリスト=万能の優等生。
- **口癖の連発はしない**（テンプレフレーズで安く作らない）。
- **各キャラの口調・口癖は、そのキャラの立ち絵を作るとき同時に確定**し、キャラクターシートに焼く（＝原稿スキルの参照正本。上記「画風の確定」末尾を参照）。

### 一人称・二人称・register の方針（温度のみ。割り当ての正本は character_bible.md）

> **一人称・二人称・register（語尾・話し方）・弁別のキモの割り当て表は `../02_characters/character_bible.md` の口調表を唯一の正本とする。** ここでは温度（声の作り分け）の方針だけを述べ、割り当ての値表は持たない。脚本も bible の表に従う。

声の作り分けは「温度の相対ズラし」だけでは足りない（初稿でカンティア・デカリス・スピノが全員「私」＋全員「お前」になり、同じ“厳しい先生”に聞こえた）。**一人称は被りをなるべく無くし、被るときは語尾・二人称・温度で弁別する。** 10人を完全に別一人称にするのは不自然なので、共有は許し**口調で差を立てる**（kako-jun「なるべく」方針）。せおの一人称は「**ボク**」（カタカナ・kako-jun 確定 2026-06-23）＝主人公専用で住人は使わない。中性（男子断定でない・Aカップに見えなくもない体つき）と噛み合うボーイッシュな中性一人称。住人の「僕」（漢字）とは表記で別物。
