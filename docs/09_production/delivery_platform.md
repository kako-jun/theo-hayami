# 提供形態 — Web サイト × name-name 埋め込み（2026-06-19 確定）

会話劇なので、自前で描画基盤を作らず **name-name（Markdown 原稿 → PixiJS ノベルプレイヤー）を流用**する。theo-hayami は「name-name プレイヤーを埋め込んだ Web サイト」になる。

## 位置づけ：ゲームにしない／ショート動画と等価（2026-06-24 kako-jun 確定）

- **ゲームにしない（意図的）。** ゲームは抵抗感が強いと予想されるため、提供形態は「**サイトの複数箇所に PixiJS（＝name-name プレイヤー）を埋め込んだ読み物**」にする。攻略・勝敗・操作を負わせない。
- **1本の脚本MD ＝ 電車の駅間で読める量。** 縦長レイアウト前提。
- **ショート動画と等価。** 縦長ゆえそのまま**自動送りのショート動画**にしやすく、自動送り動画でも**まったく同じ情報**が得られる（読む／観るのどちらでも完結する二経路。動画は studio-tatenami の声＋ studio-yokonami の絵で組む）。
- **イメージビジュアルの用途。** 集合の耽美キービジュアル（studio-yokonami #14）と各キャラの情景ビジュアル＝読書・行動（#15）は、**PixiJS 埋め込み各所のタイトル絵／ショート動画のタイトル絵**として使う。キャラビジュアルと耽美な美術が本作の売り。

## コンテンツの単位（2種類の MD）

すべて**作り置きの脚本MD**（ライブ生成なし＝引用の正確さを人の目で担保）。MD は2種類：

- **A：〔お題 × 住人〕** … 自由行動パートの独立エピソード。**順不同・自己完結**。進捗管理ゼロで成立。（→ `script_format.md`）
- **B：本筋** … 四幕の物語を進める MD。（→ `03_story/structure_hybrid.md` / `four_act_structure.md`）

**脚本の形式（2026-06-22 確定・形式X）**: A も B も **name-name 原稿形式そのもの**で書く（独自記法 `[話者]` は廃止＝変換器なし・パーサは name-name の1つ＝単一情報源）。詳細は `script_format.md`。

## 役割分担

- **会話劇本体 ＝ name-name プレイヤー**（ステートレス。渡された MD を1本再生して終わり）
- **その外側 ＝ お題選択／回遊 UI**。当面は **エントリ `script.md` の「ハブ」シーン（選択肢メニュー）**で実現する（name-name のマルチMD再生＝各お題は別 MD、ハブの選択肢 `→ シーンID` でファイル横断ジャンプ。#285。theo-hayami は name-name にプロジェクト登録済み・`scriptsDir: content/scripts`）。theo-hayami 独自サイトは後回しでよい

## サイト（置き場）の方針

- 全 MD（A も B も）をサイトに**並べる**だけ。**順序ロジックなし・ゲートなし・状態なし**。
- どこからでも開ける。**いきなり最後の本筋を読むのも自由**（本に近い）。これはエンディング「この物語で語られたことを、まず疑え／黒幕はいない」とも一致する＝読者を順番で*操作*してネタバレ箱を守る作りにしない。兄弟の osaka-kenpo と同じ「好きなところから引く」触り方。
- スピノ消失の喪失は、順に歩いた読者にちゃんと刺さる。**刺さりを強制しないだけ**。
- 任意の親切（今決めない）：ゲートはしないが**推奨の並び順は見せる**ことはできる（"guide, don't gate"）。

- **ファイル名に通し番号を付けない**（A は順不同だから）。ファイル名は永久キー `お題__住人.md`。並び順が要るならファイル名でなく別データ（manifest）で持ち、今は作らない。B（本筋）だけ四幕の実順序があるので `act{n}-{nn}` と番号を振る。

## name-name プレイヤー：ショーケース（読み専）モード

name-name に `mode: "showcase"`（仮）の1フラグを足し、書き込み系を殺して読み味だけ残す。**theo-hayami 専用の捨て実装にしない**——「他人が自作ノベルを公開する読み専モード」として name-name 本体の資産になる。

| 機能 | ショーケースモード | 理由 |
|---|---|---|
| 暗転・場面転換・立ち絵表情変更 | **ON** | 会話劇の本体 |
| BGM/SE（将来ボイスも） | **ON** | studio-tatenami が効く層 |
| バックログ／前のセリフに戻る | **ON** | 読み返しは読書体験の一部 |
| セーブ／ロード（＝name-name の Git コミット） | **OFF** | 作り置き・読み専。ユーザー単位の永続化は不要 |
| エディタ導線・キャンバス編集 | **OFF** | プレイヤーだけ切り出す |
| プロジェクト一覧・切替 | **OFF** | 1作品に固定 |
| 「続きから」（進捗の永続化） | **OFF** | MD は独立・自己完結。どこまで読んだかは管理しない＝完全ステートレス |

## 会話の描画スタイル — `dialog_style: adv | novel`（2026-06-22 確定 / 改訂）

name-name の描画スタイルは frontmatter `dialog_style` で選ぶ。値は **`adv`（下部 ADV 箱）** と **`novel`（全画面ノベル＝ToHeart 式）** の2つ。

- **2スタイルは対等。優劣・主従はない。デフォルト値という概念を持たせない**（作品ごとに明示指定する）。「ADV が本来で novel が特別」という後方互換の発想を取らない。むしろ novel を基本格に置きたいくらい、の温度。
  - 実装メモ：未指定の既存作品は `adv` 描画にフォールバックさせる（壊さないため）が、それは「対等な2択のうち未指定時の挙動」であって「正規のデフォルト」ではない。値は `novel`（「サウンドノベル」はチュンソフトの登録商標なので避ける）。
- ON/OFF は **per-game（frontmatter で1回設定）**。シーンごとのトグルにしない。theo-hayami は `novel` 固定。

### novel スタイルの中身

- **全画面背景の上に、セリフ表示中だけ半透明の黒スクリムを重ね、白文字を置く**（ToHeart 式）。name-name の既存「枠なし白文字＋ドロップシャドウ」に、新規の全画面スクリムを足す。
- **表情変化・場面転換のとき、スクリムが一瞬・滑らかに引いて立ち絵/背景を見せ、また戻る**（＝見せ場で絵を見せる。当時の Leaf 作品の手触り）。これは**エンジンが自動でやる演出**で、作者は触らない。
- **話者名ラベルは出さない**（name box OFF）。話者タグ `[…]` はソースのメタ情報（立ち絵・将来の声の制御）で、画面には名札を漏らさない。読者は立ち絵＋声で話者を判別＝小説的。せお＝読者の分身ゆえ自分のセリフに名札を付けない。脚本MD 側の扱いは `script_format.md`。
- **フォントサイズは1種類**（adv と別に持たない・切り替えない）。縦長スマホで改行が増えるのは adv も同じ＝novel 固有の問題ではない。
- **地の文（ナレーション）は今は入れない。** せおの一人称リアクションで小説的手触りを作る。必要になれば「枠なし＝ナレ風」で後付け。

### ページ送り（改頁）

- **改頁は文の境界で行う**。改頁前の最後の文が**きりよく収まる**こと（文の途中で切らない）。
- **1ページの分量は固定しない**。1〜2文でも3文以上でもよい。**長短のバランス**を取る（長文ばかり・短文ばかりはユーザーが飽きる）。画面下端まで機械的に詰めない。
- **クリック＝改頁**（次の文の追記ではなく、ページが切り替わる）。**「改頁した」と一目でわかる**視覚的な切替にする（インクリメンタル表示にしない）。

### 立ち絵：配置と話者表示（2026-06-22 追記）

名札を出さない代わりに、**誰が喋っているかを立ち絵で示す**。

- **話者が交代したら、その人の立ち絵が少しポーズ変化**して「今この人」を示す（表情変化フックに相乗り＝スクリム自動退避も連動）。
- **1文ごとに話者を入れ替えるテンポの速い言い合いはやらない**（脚本側の作法）。ゆえにポーズ変化は乱発されない。
- **配置ルール**：縦位置は**全員共通のベースライン固定**。横位置は中央でなく**左右に振る（比率配置）**。**左右は役割で決める＝質問役（せお）と回答役（住人）**。司会ヴィンチアの定位置は別途詰める。
- **素材は既存の theo-hayami シートから作る**：`docs/07_visual/reference/01_character-designs/`（全10人 character-sheet＋card、せおは fullbody/turnaround/**expressions**/costume-props）。ポーズ差分・赤らめ顔(#9)は既存シートの加工で足りる公算（新規 LoRA 必須ではない）。LoRA #6 は同一人物の再生成用。

## シークバー仕様（ネタバレ防止レビューバー）

戻る手段としてシークバーは**置く**。ただしネタバレを設計で塞ぐ：

- **ホバー中だけ表示**（普段は隠す）。
- **右端＝既読のフロンティア（＝今）**。総尺は描かない → *残量すらネタバレしない*。バーは「物語全体の物差し」でなく「自分が踏破した足跡の物差し」。
- ドラッグ可能範囲は **［先頭 〜 フロンティア］だけ**。未読側へは進めない。
- 読み進めるとフロンティアが伸び、バーが再スケール（右端は常に「今」に貼り付く）。
- **フロンティアは MD 単位**（1本の MD＝1本の足跡）。新しい MD を開いたらバーはゼロから。MD をまたいだ足跡は持たない。
- **戻ったあとの挙動：(A) そこから読み直し**（再生ヘッドが戻った位置へ移り、フロンティアまでは再読、追い越したら新規としてバーがまた伸びる）。※「覗くだけで今に戻る」(B) ではない。

## name-name 側の実装課題（別途 Issue 化）

1. プレイヤーを**読み取り専用の埋め込みウィジェット**として切り出す（現状 embed/iframe 口なし）。
2. `mode: "showcase"` フラグで上記 OFF 群を消す。
3. ネタバレ防止シークバー（ホバー表示・フロンティア=右端・未読方向ロック）。
4. **`dialog_style: novel`（全画面ノベル＝ToHeart 式）描画**（name-name #283）：セリフ中の半透明黒スクリム＋白文字、表情変化/場面転換でのスクリム自動退避、名札OFF、1フォント、文境界での改頁（最後の文がきりよく収まる・長短バランス）、改頁の明示的切替。`dialog_style` は `adv`/`novel` の対等2択でデフォルトなし。現状 name-name は下部 ADV 箱（`DialogBox.ts` 高さ180px固定）。白文字と「枠なし」モードは既存、スクリムと連動退避・改頁ロジックが新規。
5. **話者交代→立ち絵ポーズ変化＋左右配置**（別 Issue・name-name engine ＋ studio-yokonami 絵）：縦ベースライン固定・左右は役割（質問役せお/回答役住人）。素材は既存シート加工。
