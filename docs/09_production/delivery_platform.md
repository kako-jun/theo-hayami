# 提供形態 — Web サイト × name-name 埋め込み（2026-06-19 確定）

会話劇なので、自前で描画基盤を作らず **name-name（Markdown 原稿 → PixiJS ノベルプレイヤー）を流用**する。theo-hayami は「name-name プレイヤーを埋め込んだ Web サイト」になる。

## コンテンツの単位（2種類の MD）

すべて**作り置きの脚本MD**（ライブ生成なし＝引用の正確さを人の目で担保）。MD は2種類：

- **A：〔お題 × 哲人〕** … 自由行動パートの独立エピソード。**順不同・自己完結**。進捗管理ゼロで成立。（→ `script_format.md`）
- **B：本筋** … 四幕の物語を進める MD。（→ `03_story/structure_hybrid.md` / `four_act_structure.md`）

## 役割分担

- **会話劇本体 ＝ name-name プレイヤー**（ステートレス。渡された MD を1本再生して終わり）
- **その外側 ＝ theo-hayami 独自のサイト**（トップ／お題選択／回遊 UI）

## サイト（置き場）の方針

- 全 MD（A も B も）をサイトに**並べる**だけ。**順序ロジックなし・ゲートなし・状態なし**。
- どこからでも開ける。**いきなり最後の本筋を読むのも自由**（本に近い）。これはエンディング「この物語で語られたことを、まず疑え／黒幕はいない」とも一致する＝読者を順番で*操作*してネタバレ箱を守る作りにしない。兄弟の osaka-kenpo と同じ「好きなところから引く」触り方。
- スピノ消失の喪失は、順に歩いた読者にちゃんと刺さる。**刺さりを強制しないだけ**。
- 任意の親切（今決めない）：ゲートはしないが**推奨の並び順は見せる**ことはできる（"guide, don't gate"）。

- **ファイル名に通し番号を付けない**（A は順不同だから）。ファイル名は永久キー `お題__住人.md`。並び順が要るならファイル名でなく別データ（manifest）で持ち、今は作らない。B（本筋）だけ四幕の実順序があるので `act{n}-{nn}` と番号を振る。

## name-name プレイヤー：ショーケース（読み専）モード

name-name に `mode: "showcase"`（仮）の1フラグを足し、書き込み系を殺して読み味だけ残す。**theo-hayami 専用の捨て実装にしない**——「他人が自作ノベルを公開する読み専モード」として name-name 本体の資産になる。

| 機能 | ショーケースモード | 理由 |
|---|---|---|
| 暗転・場面転換・立ち絵表情変更 | **ON** | 会話劇の本体 |
| BGM/SE（将来ボイスも） | **ON** | studio-tatenami が効く層 |
| バックログ／前のセリフに戻る | **ON** | 読み返しは読書体験の一部 |
| セーブ／ロード（＝name-name の Git コミット） | **OFF** | 作り置き・読み専。ユーザー単位の永続化は不要 |
| エディタ導線・キャンバス編集 | **OFF** | プレイヤーだけ切り出す |
| プロジェクト一覧・切替 | **OFF** | 1作品に固定 |
| 「続きから」（進捗の永続化） | **OFF** | MD は独立・自己完結。どこまで読んだかは管理しない＝完全ステートレス |

## 会話の描画スタイル — 全画面オーバーレイ（ToHeart 式）（2026-06-22 確定）

下部 ADV ウィンドウ（固定の狭い箱）は**使わない**。長文が多いため、テキストに画面の高さをフルに使う。

- **全画面背景の上に、セリフ表示中だけ半透明の黒スクリムを重ね、白文字を置く**（ToHeart 式）。name-name の既存「枠なし白文字＋ドロップシャドウ」に、新規の全画面スクリムを足す。
- **表情変化・場面転換のとき、スクリムが一瞬・滑らかに引いて立ち絵/背景を見せ、また戻る**（＝見せ場で絵を見せる。当時の Leaf 作品の手触り）。これは**エンジンが自動でやる演出**で、作者は触らない。
- **これを showcase モードの既定の見た目とする。**
- ON/OFF は **per-game の既定（frontmatter で1回設定）**であり、シーンごとのトグルではない。theo-hayami は常時 ON。
- **フラグを消せない理由**：name-name は汎用プレイヤーで、他作品は下部 ADV 箱が既定。スクリムを無条件 ON にすると他作品が壊れる。フラグは「**他作品の既定を守る**後方互換スイッチ」であって、theo-hayami が操作する switch ではない。
- **話者名ラベルは出さない**（name box OFF）。話者タグ `[…]` はソースのメタ情報（立ち絵・将来の声の制御）で、画面には名札を漏らさない。読者は立ち絵＋声で話者を判別＝小説的。せお＝読者の分身ゆえ自分のセリフに名札を付けない。脚本MD 側の扱いは `script_format.md`。
- **地の文（ナレーション）は今は入れない。** せおの一人称リアクションで小説的手触りを作る。必要になれば「枠なし＝ナレ風」で後付け。

## シークバー仕様（ネタバレ防止レビューバー）

戻る手段としてシークバーは**置く**。ただしネタバレを設計で塞ぐ：

- **ホバー中だけ表示**（普段は隠す）。
- **右端＝既読のフロンティア（＝今）**。総尺は描かない → *残量すらネタバレしない*。バーは「物語全体の物差し」でなく「自分が踏破した足跡の物差し」。
- ドラッグ可能範囲は **［先頭 〜 フロンティア］だけ**。未読側へは進めない。
- 読み進めるとフロンティアが伸び、バーが再スケール（右端は常に「今」に貼り付く）。
- **フロンティアは MD 単位**（1本の MD＝1本の足跡）。新しい MD を開いたらバーはゼロから。MD をまたいだ足跡は持たない。
- **戻ったあとの挙動：(A) そこから読み直し**（再生ヘッドが戻った位置へ移り、フロンティアまでは再読、追い越したら新規としてバーがまた伸びる）。※「覗くだけで今に戻る」(B) ではない。

## name-name 側の実装課題（別途 Issue 化）

1. プレイヤーを**読み取り専用の埋め込みウィジェット**として切り出す（現状 embed/iframe 口なし）。
2. `mode: "showcase"` フラグで上記 OFF 群を消す。
3. ネタバレ防止シークバー（ホバー表示・フロンティア=右端・未読方向ロック）。
4. **全画面オーバーレイ描画（ToHeart 式）**：セリフ中の半透明黒スクリム＋白文字、表情変化/場面転換でのスクリム自動退避、per-game 既定フラグ（既定は従来の ADV 箱＝後方互換）。現状 name-name は下部 ADV 箱（`DialogBox.ts` 高さ180px固定）。白文字と「枠なし」モードは既存、スクリムと連動退避が新規。

## 最小プロト（次の一手）

脚本MD 1本 → name-name プレイヤーをブラウザに read-only で出す（設計図の既存タスクと同一）。
