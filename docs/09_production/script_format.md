# 脚本MD形式と振り付け

## 形式は name-name 原稿そのもの（2026-06-22 確定・形式X）

theo-hayami は描画を name-name に流用するので、**脚本MD は最初から name-name の原稿形式で書く**（独自記法 `[話者]` は廃止＝変換器を持たない・パーサは name-name の1つだけ＝単一情報源）。形式の正本は name-name 側 `docs/spec/markdown-v0.1.md`。要点:

- 先頭に frontmatter。**各セル MD は `engine: name-name`（worker フィルタ必須）と `title`（索引用）の2行のみ**（#48）。描画設定はセルに書かない。
- **描画設定はエントリ `script.md` に1回だけ書く**。theo-hayami の novel 既定 frontmatter（`aspect_ratio: "9:16"` / `dialog_style: "novel"` / `font_family: "Hina Mincho, serif"` / `font_size: 26` / `protagonist: "せお"` / `character_y_ratio: 1.10` / `character_fade_ms: 700` / `character_scale: 0.5` / `skip_enabled: false` / `debug_enabled: true`）は、すべて**エントリ `script.md` 側の設定**。name-name のマルチ MD 再生では描画設定はエントリ `script.md` の frontmatter からしか読まれず、各セル MD の描画設定 frontmatter は**無視される**（`?scene=` deep-link で個別セルに飛んでも同じ）。ゆえにセルには置かない（#48 で全304セルから撤去済み）。
- 場面は `## scene-id: 見出し`
- セリフは **`**話者**:`** の次行に本文（複数行可）。話者は `せお` / `ヴィンチア` / 住人名（`カンティア` 等）
- 地の文は `> 本文`（ただし theo-hayami では今は使わない。下記）
- 選択肢は **`[選択]` ブロック内**に `- テキスト → シーンID`（ハブの回遊メニュー・各話の戻り。bare `- テキスト` はセリフに吸われる）
- **本文（台詞・地の文）の記号・英数字は全角にする**（例: `？` `！` `２＋２＝４`）。`！？` も全角で書く（`!?` は使わない）。`？` `！` の後に間をあける場合は**原稿側で半角スペースを入れる**（name-name は自動挿入しない）。会話文末は句点を落として `」` にする（`。」` にしない）。ただし frontmatter、`[背景:]`、`[選択]`、選択肢のジャンプ、話者行、`9:16` や `font_size` 等の name-name 構文は半角のまま保持する。
- **三点リーダー・余韻横棒は打ちやすい ASCII で書き、name-name が上下中央の字形に正準化する（name-name#340）**:
  - 三点リーダー → 原稿は `……`（U+2026 を2連。1つでも可）と打つ。name-name が `⋯⋯`（U+22EF・上下中央）に正準化する。
  - 余韻の横棒 → 原稿は `--`（ASCII ハイフン**ちょうど2連**）と打つ。name-name が `──`（U+2500・上下中央）に正準化し、**文送り境界**にもなる。
  - この2つは正当な literal 表示用途が無いので name-name が自動変換する。`？` `！` `半角スペース` は literal 用途があるので name-name は触らない＝**原稿側で正しく書く**。
  - 経緯: #340 以前は「verbatim 描画（原稿が正本・name-name#356）」前提で `⋯⋯`/`──` を直打ちしていた。#340 で name-name が parse 時に本文を正準化するようになったため、原稿は打ちやすい `……`/`--` に統一した（既存脚本 free/ 296本・main/おはこ 8本・script.md をすべて一括変換済み）。
- **親子関係・区切りの横棒 `─`（U+2500 単独）は余韻横棒ではない別用途。正準化しない＝原稿にそのまま `─` と書き `--` にしない**。シーン見出しの `テーマ ─ 下位`（例 `## netami-menu: 人がうらやましい ─ 誰に聞く`）等がこれ。**単独 `─`（関係）と 2連 `--`→`──`（余韻）を混同しないこと**（この取り違えで一括変換が見出しを壊しかけた）。
- **その他の横棒族は本文に使わない**: em dash `—`(U+2014)・horizontal bar `―`(U+2015)・en dash 等は書かない（name-name は変換しないので割れる）。長音符（伸ばし棒）だけは `ー`(U+30FC)。シーンID・アセットパス・`→ ジャンプ先` の ASCII ハイフン `-`(U+002D 単独) は name-name 構文なので据え置く（本文の `--` **2連ちょうど**だけが余韻横棒として正準化される）。

1脚本MD ＝〔お題 × 住人〕1組（自由行動パートの一問）。実体は順送りの会話劇:

```
---
engine: name-name
title: "アバター 〜カンティアの問い〜"
---

## ohako-kantia: アバター / 住人: カンティア（能力:認識）

**ヴィンチア**:
せお、ひとつ聞かせろ。お前は今、絵の自分で人と喋っている。あれは--嘘の自分だと思うか？

**カンティア**:
……届いているのは、いつも「現れ」のほうだ。私はそれを『現象』と呼ぶ。
```

作り置きのみ（ライブ生成なし）。一本ずつ人の目で事実確認（特に引用）してから出す＝引用取り違えを設計で塞ぐ。

## セル冒頭の演出ブロック（背景→待機→登場→待機・2026-07-08 確定 / 2026-07-09 更新 #46）

各セルの本文は最初の話者行の前に**演出ブロック**を置いて始める（せお開幕セルを除く）:

```
[背景: shadow-library/xxx.webp]
[待機: 700]
[登場: せお (theo/<表情>, 左)]
[待機: 700]

**ヴィンチア** (vincia/observe, 右):
……
```

- 狙いは **背景（フェードイン）→ せお（フェードイン）→ 最初の話者（ヴィンチア）** を1つずつ立てること（#39/#46）。`[背景:]` の後に `[待機:]`、`[登場: せお ...]` の後にもう一度 `[待機:]` を置く。`[背景:]` `[登場:]` `[待機:]` はいずれも name-name 構文なので**半角のまま**書く（上記「本文は全角」ルールの対象外＝frontmatter や `[選択]` と同じ扱い）。
- **なぜ待機が2つ要るか**: name-name は**非同期が既定**＝連続したディレクティブは並行フェードで「同時」に見える。逐次（背景を先に立ててからせお、せおを立ててから話者）にするために `[待機:]` を挟む（#43）。
- **`[待機:]` は実時間待ちであり「フェード完了待ち」ではない**。ゆえに待機値は**フェード時間に合わせる**。**背景・立ち絵とも 700ms**（背景フェード＝エンジン既定、立ち絵フェード＝エントリ `script.md` の frontmatter `character_fade_ms: 700`。立ち絵は登場・退場・切り替えすべて同じ `character_fade_ms`）。**背景フェードを変えたら 1つ目の待機値も、立ち絵フェードを変えたら 2つ目の待機値も追随が要る**。
- **不変条件（テストで固定）**: `[登場: せお` 行の**直前も直後も** `[待機: N]`（N は正の整数）で、**2行上が `[背景:`**（背景→待機→登場せお→待機の並び）。`src/lib/scripts.seo-wait.test.ts` が `content/scripts/**/*.md` を全走査して検証する。新しいセル（未着手の `main/act*.md`・新規おはこ）を書くときも必ずこの2つの待機を入れる。入れ忘れるとこのテストが赤になる。
- **N=700 は既定値で、本番実機で見て詰める前提**。テストは具体値に依存せず `[待機: N]` の構造と N>0 だけを見るので、フェード時間を変えて 700→600 等にしても壊れない（値を変えたら待機値も揃える）。
- **せお開幕セル**（せお自身が最初の話者で `[登場: せお]` を持たないセル）は対象外＝待機不要。

## テーマ軸＝「人間の業」／本編おはこ・自由業グリッド（2026-06-26 確定）

- **テーマ軸はキャラ非依存の「人間の業／お題」**（死・嫉み・儲け・論破・嘘・怒り・矛盾・わかってても……）。旧「住人の十八番＝テーマ」は廃止（軸の混線だった）。テーマの網羅レジストリ＋差異化プロット＋全セルのファセット割当は **正本 [`combination_grid_drafts.md`](./combination_grid_drafts.md)**。テーマは8固定でなく9個以上に増やしてよい（レジストリから採る）。
- **本編 `main/` ＝ おはこ初登場**: 各住人がせおと初めて会うシーンを、その人の代名詞となる思想（おはこ）で描く。既存8脚本（旧 free の対角＝アバター/幻想/死/分類/矛盾/あたりまえ/きれいごと/わかってても）はここへ移設する。
- **自由 `free/` ＝ 業 × 住人**: 各業に住人が自分のレンズで答える。**おはこの技は本編専用 → free では別ファセット**を使う（本編↔自由のかぶりも塞ぐ）。
- **草稿は全グリッド差異化済み**（`combination_grid_drafts.md`）。ただし **本番化（完成脚本を `free/` へ起こす）は映えるセルから1本ずつ**。全セルを完成させる義務はない＝草稿で全体を持ち、良いミスマッチを優先して出す。
- **移行は段階的**: 業脚本が揃うまで、ハブ `script.md` と既存8脚本は現状維持（live playback を壊さない）。業脚本が貯まったらハブを業メニューへ差し替え、既存8本を `main/` へ移す。

## エントリとシーンID（マルチMD・2026-06-22）

- プロジェクトのエントリは `content/scripts/script.md`（ハブ）。`## hub:` シーンに **`[選択]` ブロックの選択肢メニュー**（`- お題名 → シーンID`）を置き、各お題 MD のシーンへ飛ぶ。各話末尾は `[選択]` ブロックで `- 別の問いを聞く → hub` を置いて戻す。
- **シーンID は全 .md でグローバル一意**（マルチMD再生でジャンプが横断解決・重複は先勝ち）。例: `hub` / `ohako-kantia`。

## 表示（2026-06-22 確定）

- 表示は**全画面オーバーレイ（ToHeart 式）**＝下部 ADV ウィンドウは使わない（長文が多い）。詳細は `delivery_platform.md`。
- **画面に話者名ラベルを出さない**（name box OFF・小説的）。`**話者**:` はソースの話者指定（どの立ち絵・将来どの声か）で、画面には名札を漏らさない。読者は立ち絵＋（将来）声で判別。せお＝読者の分身ゆえ自分のセリフに名札なし。
- **地の文（`> 本文`）は今は入れない。** せおの一人称リアクションが小説的手触りを担う。必要になれば後付け。

## 退屈させない＝振り付けを毎回変える

全話が「司会前振り→住人→せおなるほど」だと3本で飽きる。引き出しを組み合わせてテンプレ臭を消す:

- **始まり**：①せおの愚痴/素朴な疑問 ②ヴィンチアの挑発 ③住人がいきなり喋り出す ④今のアレ（暗示）
- **中間**：①せおが食い下がる ②ヴィンチアが補助線 ③住人が例え話 ④住人が逆質問で詰める
- **終わり**：①せおの「なるほど！」 ②ヴィンチアのメタ皮肉 ③住人の捨て台詞 ④問い返して余韻

4×4×4で64通り。**これは作り手のガイド**。各話で選んだ振り付けは**脚本MDに埋め込まない**（name-name にコメント構文が無い）。必要なら制作メモ / PR に書く。

## 命名・番号（2026-06-22 確定）

- **A（お題×住人）はファイル名に通し番号を付けない。** A は順不同だから。ファイル名は中身を表す永久キー `お題__住人.md`（例 `ikari__makiya.md`）。並び順が要るときはファイル名でなく**別データ（manifest）**で持つ＝並べ替えてもリネーム不要。今は manifest を作らない（YAGNI）。
- **B（本筋）は四幕に実順序があるので番号を振る** `act{n}-{nn}.md`（例 `act1-01.md`）。
- 規約の正本は `content/scripts/README.md`。
