# 脚本MD形式と振り付け

## コンテンツ単位

1脚本MD ＝〔テーマ × 住人〕1組（自由行動パートの一問）。実体は話者タグ付きの順送り脚本：

```
# テーマ: アバター / 住人: カンティア（能力:認識）
[せお]      えっ、絵の自分で生きるのって、嘘ついてることに…？
[ヴィンチア] まあ慌てるな。カンティア、頼む。
[カンティア] 諸君が"アバター"と呼ぶもの、私の言葉では『現象』だ…
[せお]      ほうほう…！
[ヴィンチア] (締めのひと刺し)
```

作り置きのみ（ライブ生成なし）。一本ずつ人の目で事実確認（特に引用）してから出す＝引用取り違えを設計で塞ぐ。**全グリッド（住人×全テーマ）を埋める義務はない**。映える組み合わせ＝良いミスマッチだけ作る。

## 退屈させない＝振り付けを毎回変える

全話が「司会前振り→住人→せおなるほど」だと3本で飽きる。引き出しを組み合わせてテンプレ臭を消す：

- **始まり**：①せおの愚痴/素朴な疑問 ②ヴィンチアの挑発 ③住人がいきなり喋り出す ④今のアレ（暗示）
- **中間**：①せおが食い下がる ②ヴィンチアが補助線 ③住人が例え話 ④住人が逆質問で詰める
- **終わり**：①せおの「なるほど!」 ②ヴィンチアのメタ皮肉 ③住人の捨て台詞 ④問い返して余韻

4×4×4で64通りの振り付け。

## 画面・表示（2026-06-22 確定）

name-name の埋め込みを流用した軽量ノベル形式。本編（四幕）も自由行動も同じ描画基盤に乗せる。表示は**全画面オーバーレイ（ToHeart 式）**＝下部 ADV ウィンドウは使わない（長文が多い）。詳細は `delivery_platform.md`。

- **話者タグ `[…]` はソースのメタ情報**（どの立ち絵を出すか・将来どの声か、の制御用）。**画面には話者名ラベルを出さない**（name box OFF・小説的）。読者は立ち絵＋（将来）声で話者を判別する。せお＝読者の分身ゆえ、自分のセリフに名札を付けない。
- **地の文（ナレーション）は今は入れない。** せおの一人称リアクションが小説的な手触りを担う。必要になれば name-name の「枠なし＝ナレ風」モードで後付けできる。

## 命名・番号（2026-06-22 確定）

- **A（お題×住人）はファイル名に通し番号を付けない。** A は順不同だから。ファイル名は中身を表す永久キー `お題__住人.md`（例 `avatar__kantia.md`）。並び順が要るときはファイル名でなく**別データ（manifest）**で持つ＝並べ替えてもリネーム不要。今は manifest を作らない（YAGNI）。
- **B（本筋）は四幕に実順序があるので番号を振る** `act{n}-{nn}.md`（例 `act1-01.md`）。
- 規約の正本は `content/scripts/README.md`（#1 で作成）。
