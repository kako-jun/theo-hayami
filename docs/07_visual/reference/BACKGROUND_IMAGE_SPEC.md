# 背景画像仕様

舞台シートから切り出した背景は、構図検討用の参照画像であり、最終素材ではない。採用候補になった背景は、必ず単体で高解像度再生成する。

## 理由

- シート内の小さい絵は解像度が足りず、name-name 埋め込み時に拡大で荒れる。
- 文字、立ち絵、UI を重ねる余白の確認ができない。
- 背景単体で再生成しないと、照明・奥行き・視線誘導を詰められない。

## 最低解像度

| 用途 | 目安 |
|---|---|
| 縦長ショート・宣伝用 | 1536 x 2732 以上 |
| 横長サイト・資料用 | 2732 x 1536 以上 |
| name-name 背景 | 1920 x 1080 以上 |
| 汎用マスター | 長辺 3000px 以上 |

## 作成順

1. シート画像を `00_source-sheets/` に保存する。
2. 構図候補を `02_stage-designs/` に切り出す。
3. 採用候補だけを、同じ構図で単体高解像度再生成する。
4. 再生成版を同じ舞台フォルダに `_hires` 付きで保存する。
5. name-name に載せる前に、立ち絵・本文・UI の重なりを確認する。

## 命名

```text
YYYYMMDD_shadow-library_grand-reading-room_ref_v01.png
YYYYMMDD_shadow-library_grand-reading-room_hires_v01.png
```

`ref` はシート切り出し、`hires` は単体高解像度生成。
