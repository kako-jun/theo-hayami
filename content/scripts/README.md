# 脚本MD の置き場所と規約（正本）

theo-hayami の会話劇は、すべて**作り置きの脚本MD**で持つ（ライブ生成なし＝引用の正確さを人の目で担保する）。
これらの MD を name-name プレイヤー（showcase モード）で再生する。エントリ `content/scripts/script.md`（ハブ）から各お題の MD へ、選択肢でジャンプして回遊する（マルチMD再生・name-name #285）。

## 置き場所

- **`script.md`（エントリ＝ハブ）**: `## hub:` シーンに **`[選択]` ブロックの選択肢メニュー**（`- お題名 → シーンID`）を置く。name-name はこの `script.md`（basename 一致）をエントリに選び、ここから各お題 MD のシーンへジャンプする。
- `free/`: **A：〔業 × 住人〕の独立エピソード**（テーマ軸＝人間の業＝死/嫉み/儲け/論破/嘘/怒り/矛盾/わかってても……）。順不同・自己完結。各話の末尾に `[選択]` ブロックで `- 別の問いを訊く → hub` を置いて戻す。
- `main/`: **B：本筋（四幕）＋ おはこ初登場**。四幕の物語を進める MD と、各住人の初登場シーン（おはこ＝その人の代名詞となる思想）。本編は `act1-01`〜`act4-04` の16本＋ `ohako-{住人}.md` 8本で完成済み。`/story` では読了ゲート付き本編ボタンとして順次解放する。
- `current/`: **C：けふのティータイム**。現代的・時事的な悩みを3人対談にする継続更新枠。自由行動の分母には混ぜない。
- `current-drafts/`: **未公開ティータイムのストック**。`src/data/teaTime.ts` に登録しない限り、一覧にも個別URLにも出さない。
- **おはこの技は本編（main/）専用** → free では各住人は別ファセットで答える。差異化プロット＋全セル割当は [`../../docs/09_production/combination_grid_drafts.md`](../../docs/09_production/combination_grid_drafts.md)。

name-name 側はプロジェクト設定 `scriptsDir: content/scripts` で `content/scripts/` 直下＋サブ1段の `.md` を全て読み、全シーンを横断ジャンプ索引に載せる（#285）。

## 命名

- **A（業×住人）＝ `業__住人.md`**（ローマ字スラッグ。**通し番号は付けない**＝順不同だから）。
  - 例: `ikari__makiya.md`（業＝怒り / 住人＝マキヤ）
  - ファイル名は中身を表す**永久キー**。並べ替えてもリネーム不要にするため、並び順はファイル名で持たない。
- **B（本筋）＝ `act{n}-{nn}.md`**（例: `act1-01.md`）。
  - 四幕には**実順序がある**ので番号を振る。

並び順が要るときは**ファイル名でなく将来の manifest（別データ）**で持つ。今は manifest を作らない（YAGNI）。

## 形式は name-name 原稿そのもの（形式X）

脚本は **name-name の原稿形式**で書く（独自記法 `[話者]` は廃止＝変換器なし・単一パーサ）。形式正本は name-name `docs/spec/markdown-v0.1.md`。要点:

- 先頭に frontmatter。**各セル MD は `engine: name-name`（worker フィルタ必須）と `title`（索引用）の2行のみ**（#48）。
  - **描画設定（`aspect_ratio` / `dialog_style` / `font_family` / `font_size` / `protagonist` / `character_y_ratio` / `character_fade_ms` / `event_image_fade_ms` / `character_scale` / `skip_enabled` 等）はエントリ `script.md` に1回だけ書く**。name-name のマルチ MD 再生では描画設定はエントリ `script.md` の frontmatter からのみ読まれ、各セル MD の描画設定 frontmatter は**無視される**（`?scene=` deep-link で個別セルに飛んだときも同じ）。だからセルに書くと冗長なので置かない。
- 場面は `## scene-id: 見出し`
- セリフは **`**話者**:`** の次行に本文。話者は `セオ` / `ヴィンチア` / 住人名（`カンティア` 等）
- 地の文は `> 本文`（theo-hayami では今は使わない）
- **選択肢は `[選択]` ブロック内**に `- テキスト → シーンID`（bare な `- テキスト` 行は選択肢にならずセリフの続きとして吸われる）

## セル冒頭の演出ブロック（背景→表示完了待機→登場→表示完了待機）

各セルの本文は、最初の話者行の前に**演出ブロック**で始める（セオ開幕セルを除く）:

```
[背景: shadow-library/xxx.webp]
[待機: 表示完了]
[登場: セオ (theo/<表情>, 左)]
[待機: 表示完了]

**ヴィンチア** (vincia/observe, 右):
……
```

- 狙いは **背景（フェードイン）→ セオ（フェードイン）→ 最初の話者（ヴィンチア）** を1つずつ立てること（#39/#46/#50）。
- **なぜ待機が2つ要るか**: name-name は**非同期が既定**＝連続したディレクティブは並行フェードで「同時」に見える。`[登場:]` は複数キャラ同時フェードインのため非同期のまま維持し、必要地点だけ `[待機: 表示完了]` で合流する。1つ目の待機で背景のロード・フェード完了を待ってからセオを登場させ、2つ目の待機でセオの表示完了を待ってから最初の話者を出す。
- **`[待機: 表示完了]` は name-name #412 への依存**。name-name 本番が #412 対応済みであることを公開条件にする。固定ミリ秒待機では初回ロード時の背景/立ち絵ロードやフェード完了を保証できないため、セル冒頭のセオ登場ブロックでは旧来の700ms待機を使わない。
- **不変条件（テストで固定）**: `[登場: セオ` 行の**直前も直後も** `[待機: 表示完了]` で、**2行上が `[背景:`**（背景→表示完了待機→登場セオ→表示完了待機の並び）。`src/lib/scripts.seo-wait.test.ts` が全セルを走査して検証する。新しいセル（未着手の `main/act*.md`・新規おはこ）を書くときも必ずこの2つの待機を入れる。
- **一般ルール**: `[待機: 表示完了]` は「次の台詞へ進む前に、直前の視覚演出を段階として見せたい境界」にだけ置く。立ち絵のポーズ変更・表情変更のたびに機械的には入れない。複数キャラを同時に出したい場合は `[登場:]` を連続で書いてから最後に1回だけ `[待機: 表示完了]` を置く。
- **セオ開幕セル**（セオ自身が最初の話者で `[登場: セオ]` を持たないセル）は対象外＝待機不要。
- `[背景:]` `[登場:]` `[待機:]` は name-name の既存ディレクティブ（形式正本は name-name `docs/spec/markdown-v0.1.md`）。content に書くだけでパーサ変更は不要。

## シーンID（全MDでグローバル一意）

`## scene-id:` の id は**プロジェクトの全 .md でユニーク**にする（マルチMD再生でジャンプが横断解決し、重複は先勝ち＝後方が到達不能になるため）。お題ごとに分かる id（例 `ohako-kantia`）、ハブは `hub`。

## 話者の表示

- `**話者**:` はソースの話者指定（どの立ち絵・将来どの声か）。
- **画面には話者名ラベルを出さない**（ToHeart 式・小説的。name-name の name box OFF）。
  読者は**立ち絵＋（将来）声で話者を判別**する。セオ＝読者の分身ゆえ、自分のセリフにも名札を付けない。

## 地の文（ナレーション）は今は入れない

- セオの**一人称リアクション**が小説的な手触りを担う。
- 必要になれば name-name の「枠なし＝ナレ風」モードで後付けできる。今は使わない。

## 方針

- **作り置きのみ**（ライブ生成なし）。一本ずつ人の目で事実確認（特に引用）してから出す＝引用取り違えを設計で塞ぐ。
- **草稿は全グリッド差異化済み**（`combination_grid_drafts.md`）。公開済みの `free/` は本番脚本として扱い、新規セルを足す場合も草稿の差異化方針を確認してから起こす。
- Codex/Claude で脚本を追加・修正する場合は freeza の `/theo-hayami-write` を入口にし、対象に応じて `main` / `free` / `tea-time` を明示する。

## 振り付け（テンプレ回避）

退屈させないため、振り付け（始まり／中間／終わりの引き出し）を毎回変える。
**4×4×4 の詳細は [`../../docs/09_production/script_format.md`](../../docs/09_production/script_format.md) を参照。**
**これは作り手のガイド。** 各話で選んだ振り付けは**脚本MDに埋め込まない**（name-name にコメント構文が無い）。必要なら制作メモ / PR に書く。
